# Tech Watch Tracker 포트폴리오 브리프

이 문서는 `Tech Watch Tracker` 프로젝트를 포트폴리오로 정리하기 위해 다른 AI 또는 작성자에게 전달하는 요약 자료입니다.

## 1. 프로젝트 한 줄 소개

Tech Watch Tracker는 경쟁사 릴리즈, RSS 피드, 기술 키워드 뉴스를 자동 수집하고 AI로 요약하여 전략 보고서와 Discord 알림으로 연결하는 로컬 웹 대시보드입니다.

## 2. 프로젝트 배경

솔루션전략팀 업무에서는 경쟁사 제품 릴리즈, 기술 트렌드, 시장 변화 신호를 반복적으로 확인해야 합니다.
기존 방식은 사람이 직접 검색하고, 기사 내용을 읽고, 필요한 내용을 따로 정리해야 해서 시간이 많이 들고 누락 가능성도 있었습니다.

이 프로젝트는 이러한 반복 리서치 업무를 자동화하고, 팀이 공동으로 활용할 수 있는 기술/경쟁사 정보 자산을 축적하기 위해 만들었습니다.

## 3. 핵심 운영 원칙

개인별 관심사는 프로필로 분리하고, 수집 데이터와 분석 산출물은 팀 자산으로 중앙 축적합니다.

팀원은 각자 관심 키워드와 폴더를 운영할 수 있고, 누적된 데이터는 전략 보고서뿐 아니라 경쟁사 비교, 제품 포지션 비교, 기술 채택 흐름 분석에도 활용할 수 있습니다.

## 4. 주요 기능

### 4.1 경쟁사 릴리즈 및 RSS 피드 수집

- GitHub Changelog, GitLab Releases 등 RSS/Atom 피드를 등록할 수 있습니다.
- 등록된 피드에서 새 릴리즈나 변경사항을 수집합니다.
- 중복 제목은 다시 저장하지 않도록 처리했습니다.
- 발행일과 수집일을 구분해서 저장하고 화면에 표시합니다.

### 4.2 기술 키워드 뉴스 모니터링

- DevOps, DevSecOps, Platform Engineering, 금융권 IT 등 관심 키워드를 등록할 수 있습니다.
- Google News RSS 기반으로 관련 뉴스를 검색합니다.
- 최근성 기준을 적용해 오래된 기사가 무분별하게 알림으로 발송되지 않도록 개선했습니다.

### 4.3 AI 요약

- 수집된 기사와 릴리즈 내용을 Gemini 기반 AI가 요약합니다.
- 카드에는 `AI 요약` 영역을 두어 핵심 내용을 빠르게 확인할 수 있게 했습니다.
- 단순 번역이 아니라 전략팀 관점에서 중요한 의미를 파악할 수 있도록 요약 구조를 조정했습니다.

### 4.4 Discord 알림

- 새 경쟁사 업데이트 또는 트렌드 뉴스가 수집되면 Discord Webhook으로 알림을 보냅니다.
- 발행일 기준 최신 항목만 알림을 보내도록 조정했습니다.
- 알림 카드에는 키워드, 출처, 발행일을 함께 표시합니다.

### 4.5 주간/월간 전략 보고서

- 누적 데이터를 기반으로 주간 또는 월간 전략 보고서를 생성할 수 있습니다.
- 보고서에는 반복 등장 키워드, 경쟁사 방향성, 시장 신호, 솔루션전략팀 시사점, 다음 액션 제안이 포함됩니다.
- 수동 생성과 자동 생성 방식을 분리했습니다.
- 보고서 생성 시 특정 폴더 또는 특정 키워드 범위만 선택할 수 있게 했습니다.

### 4.6 프로필 기반 운영

- 사용자는 각자 프로필을 만들 수 있습니다.
- 프로필별로 키워드, 폴더, RSS 피드, API Key, Discord Webhook을 관리할 수 있습니다.
- 팀원 개인 관심사는 프로필 단위로 분리하고, 데이터는 팀 자산으로 축적하는 방향으로 설계했습니다.

### 4.7 자동 스캔 스케줄러

- 웹 서버가 켜져 있는 동안 프로필의 스캔 주기 설정을 기준으로 자동 스캔을 실행하도록 개선했습니다.
- 기본 스캔 주기는 24시간입니다.
- 마지막 수집 시각과 마지막 시도 시각을 기준으로 불필요한 반복 실행을 방지합니다.

### 4.8 사용설명서 화면

- 첫 화면 사용설명서에 솔루션전략팀 관점의 활용 목적을 추가했습니다.
- 단순 보고서 작성 도구가 아니라 경쟁사 비교자료, 제품 포지션 비교, 기술 채택 흐름 분석으로 확장 가능한 도구임을 설명했습니다.
- API Key 연동, Discord Webhook 연동, 자동/수동 보고서 생성 방식에 대한 안내를 추가했습니다.

## 5. UI/UX 개선 내용

- 좌측 사이드바 기반 대시보드 구조를 유지했습니다.
- `핵심 운영 원칙` 박스를 강조 색상과 큰 글씨로 개선했습니다.
- 피드 카드에 발행일과 수집일을 분리 표시했습니다.
- 날짜는 시간/분 없이 날짜만 보이도록 정리했습니다.
- 기존 `기술적 가치 & 영향` 문구를 `AI 요약`으로 바꿔 사용자가 기능을 더 직관적으로 이해할 수 있게 했습니다.
- 전략 보고서 화면에 수동 생성, 자동 생성 설정, 폴더/키워드 기반 필터링을 추가했습니다.

## 6. 기술 스택

- Frontend: HTML, CSS, JavaScript
- Backend: Python, FastAPI, Uvicorn
- Database: SQLite
- AI: Gemini API
- Notification: Discord Webhook
- Data Source: RSS/Atom Feed, Google News RSS
- Version Control: Git, GitHub, GitHub Desktop

## 7. 보안 및 운영상 고려

- 실제 API Key, Discord Webhook, SQLite DB는 GitHub에 올리지 않도록 `.gitignore`를 설정했습니다.
- 대신 `config.example.json`을 제공해 어떤 설정값이 필요한지 예시만 남겼습니다.
- GitHub 저장소는 처음에는 Private으로 운영하도록 설정했습니다.
- 팀원 공유 시 개인별 프로필과 중앙 수집 데이터의 관계를 명확히 설명하도록 했습니다.

## 8. GitHub 형상관리

프로젝트는 GitHub에 `tech-watch-assistant` 저장소로 업로드했습니다.

초기 커밋 메시지:

```text
Initial commit for Tech Watch Tracker
```

GitHub에 올라간 주요 파일:

- `agent.py`
- `web_server.py`
- `database.py`
- `monitors.py`
- `discord_notifier.py`
- `static/`
- `README.md`
- `.gitignore`
- `config.example.json`

GitHub에 올리지 않은 민감 파일:

- `config.json`
- `tech_watch.db`
- 로그 파일
- `.env`
- `.app` 번들

## 9. 포트폴리오 작성 시 강조할 점

이 프로젝트는 단순히 웹 화면을 만든 것이 아니라, 실제 전략 업무 흐름을 기준으로 문제를 정의하고 자동화한 사례입니다.

강조할 수 있는 역량:

- 비개발자 관점에서 필요한 업무 문제를 정의함
- 경쟁사/기술 트렌드 모니터링 업무를 자동화함
- AI 요약을 실제 업무 산출물로 연결함
- Discord 알림과 보고서 아카이빙을 통해 팀 공유 흐름을 설계함
- 개인 프로필과 팀 자산 축적 구조를 함께 고려함
- 로컬 웹앱을 GitHub로 형상관리하고 협업 가능한 프로젝트 형태로 정리함

## 10. 포트폴리오용 추천 제목

아래 제목 중 하나를 사용할 수 있습니다.

- AI 기반 경쟁사/기술 트렌드 모니터링 대시보드 구축
- 솔루션전략팀 업무 자동화를 위한 Tech Watch Tracker 개발
- 경쟁사 릴리즈와 기술 신호를 자동 수집하는 전략 리서치 대시보드
- RSS/뉴스/AI 요약/Discord 알림을 결합한 기술 동향 추적 시스템

## 11. 포트폴리오용 요약 문장

솔루션전략팀의 반복적인 경쟁사 리서치와 기술 트렌드 모니터링 업무를 자동화하기 위해 로컬 웹 대시보드 `Tech Watch Tracker`를 구축했습니다.
RSS 피드와 키워드 뉴스를 수집하고, AI 요약과 Discord 알림, 주간/월간 전략 보고서 생성 기능을 연결하여 단순 정보 수집을 팀의 전략 리서치 자산으로 전환하는 흐름을 설계했습니다.
