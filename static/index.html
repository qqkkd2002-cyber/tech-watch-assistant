🎯

솔루션전략팀의 기술 시장 관측소

Tech Watch Tracker는 매번 사람이 검색하던 경쟁사 릴리즈, 기술 키워드 뉴스, 시장 변화 신호를 자동으로 모으고, 반복되는 흐름을 읽어 주간/월간 전략 보고서, 경쟁사 비교자료, 제품 포지션 분석 자료로 연결하는 업무 자동화 대시보드입니다.

핵심 운영 원칙

개인별 관심사는 프로필로 분리하고, 수집 데이터와 분석 산출물은 팀 자산으로 중앙 축적합니다.

팀원은 각자 관심 키워드와 폴더를 운영하고, 누적된 데이터는 전략 보고서뿐 아니라 경쟁사 비교, 포지션 비교, 기술 채택 흐름 분석에 활용합니다.
자동화되는 업무
  • 경쟁사 릴리즈/변경사항 수집
  • 기술 키워드 뉴스 검색 및 요약
  • 중요 업데이트 선별과 보관
  • 주간/월간 전략 보고서 초안 작성
  • Discord 기반 팀 공유
효율화되는 지점
  • 자료 검색과 1차 요약 시간 절감
  • 경쟁사 업데이트 누락 감소
  • 반복 등장 키워드와 시장 신호 누적
  • 전략회의/RFP/제안서/경쟁사 비교자료 확보
  • 팀원별 관심 영역을 프로필로 분리 관리
전략적으로 얻는 것
  • 어떤 기술 키워드가 자주 등장하는지
  • 경쟁사가 어느 방향으로 움직이는지
  • AI, DevSecOps, 거버넌스 등 반복 신호가 무엇인지
  • 우리 제품과 경쟁사의 포지션 차이가 무엇인지
  • 우리 솔루션 전략에 반영할 액션이 무엇인지
🔄

상시 시장 신호 축적

업무가 생긴 뒤 급하게 검색하는 방식에서 벗어나, 평소에 경쟁사/기술 동향을 계속 쌓아두고 필요할 때 바로 분석할 수 있습니다.

📈

보고서 초안 자동화

누적 데이터를 기반으로 기술 키워드, 경쟁사 움직임, 반복 신호, 전략 시사점을 구조화해 보고서 초안 작성 시간을 줄입니다.

⚙️ 자동 백그라운드 수집 작동 방식

컴퓨터가 켜져 있고 대시보드 서버가 가동 중인 동안에는, 설정된 스캔 주기(기본 24시간)마다 백그라운드 에이전트가 자동으로 실행되어 최신 데이터를 조용히 수집해 둡니다. 프로필 설정에서 주간/월간 보고서 자동 생성을 켜면 수집 후 보고서도 자동으로 아카이빙되고 Discord 알림이 전송됩니다. 꺼두면 데이터만 쌓이고 보고서는 버튼을 눌렀을 때만 생성됩니다.

🚀 Phase 1: 초기 설정 및 자동 수집 개시
Step 1

👤 나만의 프로필 생성

대시보드 좌측 상단 드롭다운 옆의 **[+]** 버튼을 눌러 본인의 이름이나 관심 영역(예: 기획자 홍길동, 연구용 공통)으로 새로운 프로필을 생성합니다. (별도의 회원가입이나 로그인은 필요 없습니다.)

Step 2

⚙️ 프로필 설정 및 API/피드 연동

[프로필 설정] 메뉴로 이동하여 본인 프로필의 핵심 정보들을 입력해 줍니다.

  • API Key 연동: AI 분석에 사용할 본인의 Gemini API Key를 입력합니다.
  • 경쟁사 RSS 피드 추가: 수집을 추적할 경쟁사 사이트의 RSS 피드 주소(예: GitHub Changelog)를 추가합니다.
  • 추적 키워드 추가: 최신 뉴스 트렌드를 추적할 관심 키워드(예: AI, DevOps)를 추가합니다.
  • 디스코드 알림 연동: 수집이 끝났을 때 실시간 브리핑을 전송받을 디스코드 채널 웹훅 주소를 입력합니다.
  • 보고서 생성 방식 선택: 주간/월간 보고서 자동 생성을 켜면 자동 아카이빙, 끄면 수동 생성 방식으로 운영됩니다.
Step 2-1. Gemini API Key 발급/입력 방법
  1. Google AI Studio에서 Gemini API Key를 발급합니다.
  2. [프로필 설정]의 Gemini API Key 입력칸에 붙여넣습니다.
  3. 키는 외부에 공유하지 말고, GitHub/GitLab에는 올리지 않습니다.
  4. 무료 티어를 사용할 경우 호출 제한이 있을 수 있어, 앱이 적당한 속도로 분석하도록 구성되어 있습니다.
Step 2-2. Discord Webhook 연동 방법
  1. Discord에서 알림을 받을 채널의 채널 편집 메뉴를 엽니다.
  2. 연동 또는 Integrations 메뉴에서 Webhook을 생성합니다.
  3. 생성된 Webhook URL을 복사해 Discord Webhook URL 입력칸에 붙여넣습니다.
  4. Webhook URL을 아는 사람은 해당 채널에 메시지를 보낼 수 있으므로 외부 공유를 피하고, 노출되면 즉시 재발급합니다.
Step 3

▶️ 스캔 시작 (자동 백그라운드 수집 활성화)

설정이 끝나면 좌측 프로필 선택 영역 하단에 있는 [▶️ 스캔 시작] 버튼을 클릭합니다.

첫 스캔이 가동되어 실시간 수집이 개시됩니다. **이후부터는 컴퓨터가 켜져 있는 동안 설정한 주기(예: 24시간)마다 백그라운드 에이전트가 최신 정보들을 수집하여 누적**해 줍니다. 자동 보고서 옵션이 켜져 있으면 같은 주/같은 월의 보고서 중복 생성을 피하면서 주간·월간 보고서를 아카이빙합니다.

💡 Phase 2: 수집 데이터를 이용한 전략적 업무 활용 (뭘 할 수 있나요?)
Step 4

📰 실시간 피드 모니터링

자동 수집된 자료들은 [모니터링 피드] 메뉴에 실시간 요약 카드로 차곡차곡 쌓입니다. 검색어 검색 및 '경쟁사 문서' / '키워드 뉴스' 분류 필터를 통해 수집 현황을 상시 모니터링할 수 있습니다.

Step 5

★ 중요 자료 별표 등록 및 관리

피드를 훑어보다가 우리 팀 제품이나 기획 전략에 정말 중요한 단서가 되는 안건이 보인다면, 카드 우측 상단의 **별표(★) 버튼**을 클릭하여 **[중요 보관함]** 메뉴에 모아놓을 수 있습니다.

💡 모니터링 피드에서 별표를 켜거나 꺼도 피드 목록에서는 사라지지 않고 유지됩니다. 단, **중요 보관함** 메뉴 내부에서 별표를 해제하면 보관함 리스트에서만 실시간으로 즉시 제외됩니다.

Step 6

📊 AI 전략 보고서 생성 및 아카이빙

축적된 자료들을 기반으로 주간/월간 전략 보고서를 자동 또는 수동 방식으로 생성할 수 있습니다.

  • 자동 생성: 프로필 설정에서 자동 보고서 생성을 켜면, 수집 후 주간/월간 기술 신호 보고서가 자동으로 생성되어 아카이빙됩니다.
  • 수동 생성: [월간 전략 보고서] 탭 상단의 [주간 보고서 생성] 또는 [월간 보고서 생성] 버튼을 눌러 필요할 때 직접 생성할 수 있습니다.
  • 보관함 맞춤형 보고서: `중요 보관함` 메뉴 상단의 **[⭐ 중요 보관함 기반 보고서 생성]** 버튼을 클릭하면, 내가 직접 선별해 별표를 쳐놓은 핵심 안건들만 날카롭게 분석한 전용 보고서를 작성해 줍니다.
  • Discord 알림: 보고서가 생성되면 핵심 요약과 함께 Discord 채널로 알림이 전송됩니다.
Step 7

📄 보고서 내려받기 및 팀 내 의견 공유

[월간 전략 보고서] 메뉴로 이동하여 완성된 보고서를 확인하세요. 우측 상단 버튼을 통해 오리지널 마크다운(`.md`) 파일로 내려받거나, 깔끔한 **백색 A4 인쇄 친화용 PDF**로 변환하여 출력 및 소장할 수 있습니다.

💬 공유 주소(http://localhost:8000)를 통해 우리 팀원들과 대시보드를 공유하고, [소통 게시판]을 활용해 수집된 경쟁사 현황에 대한 분석 건의 및 피드백 의견을 실시간으로 나누며 협업하세요.
