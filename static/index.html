Tech Watch Tracker 소개

자동 수집으로 기술/경쟁사 소식을 계속 모으고, AI가 후보를 좁힌 뒤 사용자가 업무 신호와 학습 신호를 확정하는 편집장형 모니터링 도구입니다.

🎯

솔루션전략팀의 AI 편집장형 기술 관측소

Tech Watch Tracker는 사용자가 매번 검색하지 않아도 경쟁사 릴리즈, 기술 레퍼런스, 키워드 뉴스, 시장 변화 신호를 주기적으로 수집합니다. 쌓인 자료는 AI가 먼저 검토 대기·노이즈로 좁히고, 필요한 항목만 요약한 뒤 사용자가 업무 신호 또는 학습 신호로 확정해 보고서와 지식 성장 자료로 활용합니다.

핵심 운영 원칙

자동 수집으로 시장 신호를 쌓고, AI가 후보를 좁히고, 사람이 업무 신호와 학습 신호를 확정합니다.

모든 기사를 사람이 다 읽는 방식이 아니라, AI가 만든 검토 대기 목록을 편집장처럼 판단하는 워크플로우입니다.

편집장 판단이 쌓일수록 AI도 더 좋아집니다. 업무 신호, 학습 신호, 노이즈로 나눈 판단이 다음 후보 정리와 보고서 품질을 높이는 학습 데이터가 됩니다.
01

자동 수집

서버가 켜져 있으면 설정한 주기마다 RSS, Docs, 키워드 뉴스가 쌓입니다. 발행일과 수집일을 따로 남겨 최신성을 확인합니다.

RSS·Docs·뉴스 발행일/수집일
02

AI 후보 정리

쌓인 전체 자료를 그대로 읽지 않고, AI가 먼저 새 검토 대기 후보를 올립니다. 카드에는 시장 신호, 경쟁사, 제품 힌트, RFP 근거 같은 고정 태그가 붙습니다.

검토 대기 노이즈 추천 태그
03

필요한 후보만 요약

판단이 애매한 검토 대기 카드만 AI 요약합니다. 모든 기사에 비용을 쓰지 않고, 볼 만한 후보에 집중합니다.

AI 요약 검토 대기 5개
04

편집장 판단

사용자가 카드를 업무 신호, 학습 신호, 노이즈로 확정합니다. 이 판단이 쌓일수록 AI가 어떤 신호를 좋아하는지 배웁니다.

업무 신호 학습 신호 노이즈
05

보고서와 팀 자산화

확정된 업무 신호와 학습 신호, 요약 근거는 후보 작업대에서 빠지고, 대시보드·수집 원장·전략 보고서에서 다시 꺼내 쓰는 팀 자산이 됩니다.

보고서 팀 근거 전략 자산
⚙️ 팀원에게 설명할 한 문장

모든 기사를 사람이 읽는 앱이 아니라, AI가 후보를 만들고 사람이 업무 신호와 학습 신호를 확정하는 앱입니다. 확정된 신호와 요약 근거는 솔루션전략팀의 경쟁사·기술 신호 자산으로 누적됩니다.

사용 가이드

자동 수집으로 자료가 쌓이고, AI 후보 정리와 요약을 거쳐 사람이 업무 신호와 학습 신호를 확정하는 전체 흐름을 안내합니다.

Phase 1: 처음 접속하면 할 일
Step 1

내 프로필 선택 또는 생성

좌측 상단 프로필 선택에서 본인 프로필을 고릅니다. 없다면 + 버튼으로 새 프로필을 만듭니다. 프로필은 개인 관심사와 수집 기준을 분리하기 위한 공간입니다.

Step 2

내 관심 영역 확인

프로필 설정에서 내가 보고 싶은 키워드, 폴더, RSS 피드가 맞게 들어가 있는지 확인합니다. 예: DevOps, 보안, AI Engineering, 금융 차세대, 경쟁사 제품명.

원칙

개인 관심사는 프로필로, 결과물은 팀 자산으로

각자 관심 키워드는 다르게 운영할 수 있지만, 수집된 기사와 분석 결과는 팀이 함께 보는 자료가 됩니다. 개인별 탐색이 쌓이면 경쟁사 동향, 기술 트렌드, 보고서 작성에 활용할 수 있는 공용 지식 자산이 됩니다.

Phase 2: 관심 키워드와 수집 대상을 정리하기
Keyword

키워드 등록

내가 계속 추적하고 싶은 기술명, 제품명, 시장 키워드를 등록합니다. 키워드는 뉴스 검색과 트렌드 분석의 기준이 됩니다.

Folder

폴더로 묶기

비슷한 키워드는 폴더로 묶습니다. 예를 들어 DevOps 폴더에 CI/CD, GitOps, IDP를 넣으면 나중에 보고서 범위를 잡기 쉽습니다.

AI 추천

놓친 키워드 추천받기

AI 키워드 확장 추천을 누르면 현재 키워드를 기준으로 관련 키워드와 추천 폴더를 제안합니다. 필요한 항목은 개별 추가하거나 추천 키워드 모두 추가로 한 번에 반영합니다.

정리

중복 표기 정리

정리 제안 적용은 DevOpsit, Devsecops, CICD처럼 중복되거나 표기가 흔들리는 키워드를 더 일관된 이름으로 정리하는 기능입니다. 정리 전에는 추천 결과를 확인한 뒤 적용합니다.

Phase 3: 자동 수집으로 자료 쌓기
Auto

주기별 자동 수집을 켜두면 서버가 실행 중일 때 설정한 주기마다 RSS, 경쟁사 문서, 키워드 뉴스를 확인합니다. 자동 수집은 자료를 쌓는 단계이고, AI 요약이나 신호 확정과는 별개의 기능입니다.

좌측 지금 수동 수집은 자동 주기를 기다리지 않고 즉시 한 번 수집하는 버튼입니다. 새로운 키워드를 추가했거나 회의 전에 최신 자료가 필요할 때 사용합니다.

Raw

수집 원장 확인

수집 원장에는 수집된 원본 목록이 모두 남습니다. 발행일과 수집일을 구분해 최신성을 확인하고, 필요하면 검색이나 필터로 원문 자료를 다시 찾습니다.

Next

수집 후에는 후보 정리로 이동

수집이 끝났다고 바로 모든 기사를 읽을 필요는 없습니다. 다음 단계에서 후보 정리로 이동해 새로 올라온 검토 대기 항목만 처리합니다.

Team

Discord

Discord Webhook

팀 채널로 수집 결과나 보고서 알림을 보내기 위한 주소입니다. Webhook URL이 노출되면 해당 채널에 메시지를 보낼 수 있으므로 외부 공유를 피합니다.

Server

로컬 서버와 조직 운영

현재 localhost:8000은 서버를 실행한 PC에서 보는 주소입니다. 팀원이 항상 쓰려면 별도 서버에 올려 공통 접속 주소를 제공하는 방식이 적합합니다.

GitHub

GitHub의 역할

GitHub는 앱을 실행하는 곳이 아니라 코드 백업과 형상관리를 위한 공간입니다. 화면 변경은 로컬 파일과 실행 중인 서버에 먼저 반영되고, GitHub 푸시는 그 변경을 저장소에 올리는 단계입니다.

소통 게시판 (Discussion Board)
